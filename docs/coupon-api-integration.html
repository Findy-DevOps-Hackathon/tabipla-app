クーポン機能 API連携

tabipla-app / Team A (A-2) — 2026-06-28 作成

概要

管理画面（admin-web）で作成したクーポンが、観光者向けアプリ（user-web）にリアルタイムで表示されるようにした。 変更前はハードコードされたデモデータだったクーポンを、PostgreSQL のテーブル → REST API → React UI の一気通貫で繋いだ。

Before / After

BEFOREハードコード

  • data/spots.ts に全20件のクーポン文言を直書き
  • クーポンコードは spot ID から決定的に生成（デモ用）
  • 管理画面で追加・削除しても user-web には反映されない
  • DB テーブルは存在するが user-web とは未接続

AFTERDB連携

  • admin-web でクーポン作成 → PostgreSQL に保存
  • GET /coupons/list API でスポット名付きで取得
  • user-web の SpotDetailModal が API からリアルタイム取得
  • DB にクーポンがないスポットは静的データにフォールバック

データの流れ

管理者（自治体）が作成

admin-web
クーポン作成 → POST /admin/coupons → PostgreSQL
coupons テーブル

観光者が閲覧

user-web
スポット詳細を開く → GET /coupons/list → coupons JOIN spots
(スポット名でマッチ) → SpotDetailModal
に表示

SQL テーブル構造

SQLcoupons テーブル（PostgreSQL）

CREATE TABLE IF NOT EXISTS coupons (
  id            TEXT PRIMARY KEY,       -- UUID
  spot_id       TEXT NOT NULL           -- 対象スポット（spots.id）
    REFERENCES spots(id) ON DELETE CASCADE,
  title         TEXT NOT NULL,          -- 表示名（例: ソフトクリーム8%OFF）
  description   TEXT,                   -- 利用条件・備考
  discount_percent INTEGER NOT NULL,    -- 割引率（5〜10想定）
  created_at    TIMESTAMPTZ DEFAULT NOW() NOT NULL,
  updated_at    TIMESTAMPTZ DEFAULT NOW() NOT NULL
);
CREATE INDEX coupons_spot_id_idx ON coupons(spot_id);

フォールバック設計

DB にクーポンが登録されていないスポットでは、従来通り data/spots.ts の静的クーポン（rec.coupon）を表示する。 これにより、admin-web からクーポンを追加する前でもデモとして全スポットにクーポンが表示される。

条件表示されるクーポン
DB にクーポンありDB データ（title, description）を優先表示
DB にクーポンなし & 静的データあり静的データのクーポン文言を表示
両方なしクーポンセクション非表示

動作確認手順

  1. Docker Desktop を起動（PostgreSQL 5433, Elasticsearch 9200）
  2. backend-api を起動（ポート 3001）
  3. curl http://localhost:3001/coupons/list で JSON が返ることを確認
  4. admin-web（ポート 5174）で taro.yamada@test.com / test-admin-password でログイン
  5. 「クーポン」メニューで新規作成 → DB に保存される
  6. user-web（ポート 5173）で該当スポットを開く → 作成したクーポンが表示される
